# 03 - ARE/Gaia2/OpenClaw 入门

## 本章目标

用通俗语言解释 ARE、Gaia2、OpenClaw 分别是什么，以及它们在 ARE2Train 里承担什么角色。

## ARE 是什么

ARE 可以理解成一个 agent 环境平台。它负责提供任务世界、工具接口、环境状态和验证机制。

## Gaia2 是什么

Gaia2 是建立在 ARE 思路上的 benchmark。它模拟个人助手任务，例如：

- 查邮件。
- 改日历。
- 找联系人。
- 处理购物或预约。
- 面对歧义时向用户提问。
- 处理动态事件和时间约束。

## OpenClaw 是什么

OpenClaw 可以理解成运行 agent 的 harness / runtime。它负责把模型、prompt、工具调用和环境交互串起来。

## 三者关系

```text
ARE/Gaia2: 提供任务和环境
OpenClaw: 运行 agent
ARE2Train: 收集轨迹、构造训练数据、训练模型、评测模型
```

## 为什么 Gaia2 适合训练

Gaia2 不只是能判断最后答案对不对。它还可以检查关键动作是否正确，例如：

- 是否找到了正确联系人。
- 是否修改了正确的日历事件。
- 是否误改了无关对象。
- 是否在歧义时先问用户。

这些检查结果可以变成 reward。

## 输入

- ARE/Gaia2 数据和环境。
- OpenClaw 或等价 agent runtime。
- 一个可调用的模型。

## 输出

- 能运行一个简单 Gaia2 task。
- 能看到工具调用和 verifier 结果。
- 能理解 trace 里每个字段的意义。

## 常见坑

- 把 Gaia2 当成普通问答数据集。
- 只看最终分数，不看工具调用过程。
- 不理解环境状态被修改后如何验证。
- 忽略动态事件和时间信息。
